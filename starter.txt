USER TYPES

Create these user roles:

1. Student
2. Tutor
3. Parent
4. Admin

Save each user's role in the database.

Role definitions:

Student:
Learner using tutoring, assignments, messages, study planning, AI tutor tools, and progress tracking.

Tutor:
A human tutor who can apply to teach, create a tutor profile, manage students, receive payments, create assignments, respond to messages, and optionally bring their own students onto the platform.

Parent:
A parent or guardian who can view linked student progress, assignments, calendars, messages, payment status, and tutor information.

Admin:
The platform owner who manages users, tutor applications, subscriptions, payment settings, subjects, AI tutor assignment boards, and platform activity.

WARM WELCOME JAVASCRIPT REQUIREMENT

Welcome back, [Student Name]. Ready to learn, grow, and thrive today?

For parents:

Welcome back, [Parent Name]. Here is your student's learning progress today.

Requirements:

The welcome message can be created with JavaScript, but login security must not depend on JavaScript.
Authentication and authorization must be handled by PHP sessions and role checks.

js/welcome.js

data-user-name

Example behavior:

JavaScript reads the role and name.
JavaScript displays the correct warm welcome message.
JavaScript should support student, tutor, parent, and admin roles.
